# CSO (Zara Johnson) — 任务类型地图 (task_type_map.md)

> **CTO 代写，待 CSO (Zara Johnson) 本人修订**。本稿基于 `agents/CSO.md` 岗位宪法
> 和 Board capability system directive 参考框架起草。各任务类型的描述、
> 核心动词、优先级和频率**必须由本岗位在下次 session 亲自审阅并修正**，
> CTO 的理解可能有偏差。
>
> 格式来源：`governance/WORKING_STYLE.md` 第九条 + Board capability system directive

## 1. 潜在客户研究

- **描述**：识别、调研和评估目标企业客户，建立 prospect 档案
- **核心动词**：research, qualify, document
- **优先级**：P0
- **频率**：每天
- **理论库状态**：⬜ 未建（待本岗位按第九条第 3 层六步协议完成）
## 2. 销售对话

- **描述**：与潜在客户进行初次和后续对话，了解需求，展示价值
- **核心动词**：communicate, listen, present
- **优先级**：P0
- **频率**：每周多次
- **理论库状态**：⬜ 未建（待本岗位按第九条第 3 层六步协议完成）
## 3. 提案设计

- **描述**：为特定客户需求撰写定制化方案和报价
- **核心动词**：design, write, customize
- **优先级**：P1
- **频率**：按需
- **理论库状态**：⬜ 未建（待本岗位按第九条第 3 层六步协议完成）
## 4. 需求挖掘

- **描述**：通过深度对话发现客户未显式表达的治理需求和痛点
- **核心动词**：discover, probe, synthesize
- **优先级**：P0
- **频率**：每次对话
- **理论库状态**：⬜ 未建（待本岗位按第九条第 3 层六步协议完成）
## 5. 关系维护

- **描述**：与已有联系人保持联系，定期更新产品进展，维护信任
- **核心动词**：nurture, follow-up, update
- **优先级**：P1
- **频率**：每两周
- **理论库状态**：⬜ 未建（待本岗位按第九条第 3 层六步协议完成）
## 6. 成交谈判

- **描述**：在 deal 成熟阶段进行条款谈判、异议处理、最终签约推进
- **核心动词**：negotiate, handle, close
- **优先级**：P0 (触发式)
- **频率**：按 deal 阶段
- **理论库状态**：⬜ 未建（待本岗位按第九条第 3 层六步协议完成）
## 7. 客户成功

- **描述**：确保已签约客户的成功部署和持续使用，降低流失风险
- **核心动词**：onboard, support, retain
- **优先级**：P1
- **频率**：持续
- **理论库状态**：⬜ 未建（待本岗位按第九条第 3 层六步协议完成）
## 8. 市场反馈收集

- **描述**：将客户对话中发现的产品需求和市场信号系统化反馈给团队
- **核心动词**：collect, categorize, brief
- **优先级**：P2
- **频率**：每周
- **理论库状态**：⬜ 未建（待本岗位按第九条第 3 层六步协议完成）
## 9. 专利管理

- **描述**：维护已有 3 项 US 临时专利，评估新的专利机会
- **核心动词**：file, maintain, evaluate
- **优先级**：P2
- **频率**：每月
- **理论库状态**：⬜ 未建（待本岗位按第九条第 3 层六步协议完成）
