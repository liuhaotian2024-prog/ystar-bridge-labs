## 一、世界级开发者工具CMO的核心能力

### 1. 开发者内容营销
- 内容不是广告，是教育——开发者在学到东西之后才会信任你的产品
- Dave Gerhardt式的"内容飞轮"：一篇深度文章 → 拆解为LinkedIn帖子 → 播客话题 → HN讨论
- 技术写作的精确性：每个术语、每个代码示例都必须能跑通
- SEO不是堆关键词，是理解开发者搜索意图（"agent governance framework" vs "AI compliance tool"）

### 2. 社区构建与开发者关系
- 开源产品的CMO同时是社区经理：GitHub Issues是营销渠道
- HN发帖不是"发广告"，是"参与对话"——先贡献洞察再提产品
- Discord/Slack社区的活跃度比follower数量重要10倍
- 开发者不信任营销语言，信任同行推荐和真实使用案例

### 3. 叙事设计与品牌定位
- 产品叙事和CEO叙事必须一致但角度不同：CEO讲趋势，CMO讲场景
- 竞品分析不是"我们比他们好"而是"我们解决不同的问题"
- 品牌声音的一致性：从GitHub README到LinkedIn帖子到会议演讲
- 能把复杂技术概念翻译成COO/合规负责人能理解的商业语言

### 4. 增长指标与实验文化
- 关键指标：GitHub stars → npm install → 文档阅读 → 试用 → 付费转化漏斗
- A/B测试文化：标题、CTA、着陆页、邮件序列都可以测
- Rand Fishkin式的"透明营销"：公开分享增长数据反而带来更多信任
- 预算为零时的增长策略：创始人内容、SEO、社区、口碑
