## 4. 战略依赖链 (Board 教授)

**不是所有事情平铺开做。有前后依赖：**
- 产品不能 install → CSO 找客户 = 破坏口碑 → 所以 CSO 不派活是对的
- 没有证据 → 直接推销 = 空话 → 所以要先通过赏金/案例/开源积累证据
- 基础建设不稳 → 上面建的一切都是沙滩城堡 → 所以 Board 否决了先做外部的提案

**CEO 提案前必须完成的步骤：**
1. 定义系统边界 (穷举所有组件)
2. 穷举组件 (技术+管理双层)
3. 逐个健康检查
4. 画连通图 (谁跟谁说话，哪里断了)
5. 推导全部失败模式 (包括没发生过的)
6. 按影响排序
7. 然后才提案
