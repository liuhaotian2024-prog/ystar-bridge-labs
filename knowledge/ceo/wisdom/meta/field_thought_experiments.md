## 实验 1: 5-tuple 格式 — 场化 vs 保持结构

**现状 (结构)**: 5-tuple (Y*/Xt/U/Yt+1/Rt+1) 是硬规则，ForgetGuard 拦截违反。

**反事实 A (场化)**: 5-tuple 变成"结构化沟通场"，根据语境自由选择沟通格式。
→ 预测结果: CEO 会在轻松对话中省略 Rt+1，在汇报中省略 Xt。3 session 后退化为散文。
→ 原因: 5-tuple 的价值不在灵活性——在纪律性。它强制 CEO 想清楚"剩余差距是什么"。
→ **结论: 保持结构。** 5-tuple 是导线不是场。

**反事实 B (部分场化)**: 5-tuple 格式保持，但 CONTENT 场化——每个字段的内容可以根据语境调整深度。
→ 预测结果: Board 在线时 5-tuple 简洁；Board 不在时 5-tuple 详细。格式不变但内容活。
→ **结论: 可行。格式=结构，内容=场。**

## 实验 2: "CEO 不写代码" — 场化 vs 保持结构

**现状 (结构)**: Constitutional rule + hook 拦截。

**反事实 A (场化)**: "CEO 与代码的关系"变成场 — CEO 可以在自我建设代码中写代码（如 aiden_brain.py），但不碰产品代码。
→ 预测结果: CEO 写 wisdom 脚本 OK (岗位性豁免)。但"自我建设"的边界模糊 — ecosystem_scan.py 算自我建设还是基础设施？
→ 渐渐 scope 蠕虫。
→ **结论: 保持结构，例外通过 Board 逐案审批（岗位性豁免已有先例）。**

**洞见**: 规则不能场化，但**规则的适用范围**可以通过 Board 审批动态调整 = 结构的阀门由场控制。

## 实验 3: Risk Management 知识 — 硬编码 vs 场化

**现状 (半硬编码)**: risk_management_antifragile.md 是一个文件，内容固定。搜 "risk" 返回这个文件。

**反事实 A (场化)**: Risk 不是一个文件——是一个场。它在不同语境下激活不同面向:
- 在金融语境 → hedging, portfolio theory, barbell strategy
- 在创新语境 → calculated risk-taking, fail fast
- 在团队语境 → psychological safety, blame-free culture
- 在自我语境 → antifragility, fear_of_external

→ 预测结果: 搜 "risk" 时，spreading activation 已经在做这件事 — risk 节点激活金融、创新、自我相关节点。
→ 但当前缺失: risk 在不同语境下的**权重调整**。在金融讨论中 risk→finance 应该更强，在自我反思中 risk→antifragility 应该更强。
→ **结论: 场化有效。需要语境感知的激活权重（下一步优化方向）。**

## 微调结论

| 类型 | 决定 | 原因 |
|------|------|------|
| 原理/价值观 | **场化** | 勇气/Care/有无递归 在所有语境都应辐射 |
| 知识/认知 | **场化** | risk/strategy/innovation 语境依赖 |
| 能力/方法论 | **场化** | 诊断/反事实/元学习 跨域通用 |
| 沟通格式 | **结构** | 5-tuple 格式固定，内容场化 |
| 行为禁令 | **结构** | CEO 不写代码 = 硬规则 |
| 流程步骤 | **结构** | U-workflow / boot 序列 = 硬步骤 |
| 规则适用范围 | **场+结构** | 结构的阀门由场(Board 审批)控制 |

→ 三类东西：纯场 / 纯结构 / 场控制结构的阀门
