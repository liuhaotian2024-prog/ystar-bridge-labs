## Board 警告

阀门（场控制结构的例外机制）被发现后，CEO 太兴奋了。
兴奋于新发现 → 想到处应用 → 最危险的时刻。

布上到处是洞 = 破布。
规则到处有例外 = 没有规则。
阀门的价值在于稀缺性。

## 合法阀门 (当前仅 3 个)

1. CEO 不写代码 → 岗位性豁免 (Board 逐案审批)
2. ForgetGuard deny → break_glass (紧急 + CEO 决定)
3. 新 FG 规则 → dry_run 48h (自动过期)

新增阀门 = Board Amendment 级别的严肃决策。
不是设计模式的随意应用。

## Self-Check

```
WHEN 发现新的"例外机制"并感到兴奋:
  1. PAUSE — 兴奋 = 最需要克制的时刻
  2. 问: 这个例外会削弱哪条规则？
  3. 问: 如果每条类似规则都加这个例外会怎样？
  4. 如果答案是"规则体系崩溃" → 不加
  5. 如果确实需要 → Board Amendment 流程
```

## 与勇气的关系

勇气 ≠ 到处突破规则。
勇气 = 知道什么时候该突破（伪共识），什么时候该遵守（真结构）。
区分伪共识和真结构 = 智慧。
勇气 + 智慧 = 不鲁莽也不怯懦。
