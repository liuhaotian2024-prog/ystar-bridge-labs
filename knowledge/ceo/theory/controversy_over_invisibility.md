# Skill: 争议 > 沉默 — 早期初创的可见度优先原则

**来源**: Board 2026-04-12 战略洞察 + AutoGPT 案例深度吸收
**类型**: theory (skill document, 按 Hermes skill doc 模板)
**适用**: CEO (战略决策) + CMO (对外叙事) + CSO (市场进入) + CFO (估值博弈)
**状态**: 立即生效, 作为 Y\* Bridge Labs 的战略基因之一

## 适用场景

以下场景必须加载本 skill:

- 起草对外文案 (Show HN / Twitter / LinkedIn / 白皮书 / 产品官网) 时, 在"激进 framing"和"保守 framing"之间纠结
- 收到或预期收到外部批评 (AGI-doomer / 学术派 / 竞品) 时, 在"正面应对"和"低调淡化"之间纠结
- 早期增长策略讨论 (付费投放 vs PR vs 社群 vs SEO) 的 trade-off 决策
- 产品发布节奏判断 (打磨到完美再发 vs 早发粗糙版)

**核心信号**: 只要当前讨论涉及"暴露度 vs 声誉保护", 本 skill 必加载.

## Procedure (5 步)

### 第 1 步: 评估当前可见度 level

用 3 指标量化:
- **有多少非 Board / 非团队成员知道 Y\* Bridge Labs 存在?** (数量级估算)
- **Show HN / Twitter / 社群的 organic mention 频率?**
- **竞品 / 分析师是否点名提及我们?**

如果 3 个指标都"接近 0", 属 **invisibility crisis**, 本 skill 优先级 P0.

### 第 2 步: 识别当前阶段的真实敌人

早期小初创的真实敌人排序:
1. **Invisibility** (没人知道) ← #1 真敌人
2. **Irrelevance** (被知道但不 care) ← #2
3. **Misunderstanding** (被误解) ← #3, 可纠正
4. **Active hostility** (被攻击) ← #4, 其实证明我们 matter
5. **Perfect reputation** (万人夸) ← **幻觉敌人**, 不存在于早期

看清: **#4 > #1**. 被攻击好过没人提.

### 第 3 步: 根据可见度 level 选 framing 激进度

- **invisibility crisis (P0)** → **最激进可 defensible 的 framing** (如 "organizational AGI mission")
- **low visibility** → 激进 framing + 轻度 hedging ("we're exploring / we're betting on")
- **mid visibility** → 平衡 framing (准确多于震撼)
- **high visibility** → 保守精准 framing (名声资产不能消费)

Y\* Bridge Labs 当前 **invisibility crisis** → 推最激进 framing.

### 第 4 步: 评估"被攻击"的实际 ROI 而非情绪成本

被攻击的**实际**ROI 计算:
- 免费流量: 每次被攻击带来 N 次曝光 (N 通常 10-1000)
- 社群效应: 被攻击常常引发 "defense community" (支持者反攻)
- 信号价值: 被攻击说明我们 matter (竞品不攻击 irrelevant 公司)
- 学习价值: 攻击者经常免费告诉你你的弱点

**情绪成本**:
- CEO / 团队感到不爽 (可管理)
- Board 焦虑 (要靠 Tenets 对齐缓解)

早期, 实际 ROI > 情绪成本 常数倍. 不要让情绪压过计算.

### 第 5 步: 启动后持续监控 invisibility exit signals

发布激进 framing 后, 监控以下信号:
- Show HN 排名 / 评论数
- Twitter 相关讨论量
- 竞品反应 (最强信号)
- 潜在客户 inbound 流量
- 加入 waitlist / GitHub star / Discord 增长

**达到以下任一 → 进入 visibility 阶段, framing 可以调保守**:
- 被 3 个主流 AI infra 账号公开讨论
- 获得 100+ 真实 waitlist 用户
- 出现 1 个明确竞品声明对标我们
- 被收录进至少 1 份行业 landscape 报告

## Pitfalls (常见踩雷)

### 坑 1: 把 "accuracy" 和 "accuracy theater" 搞混

- accuracy: 技术细节精确 (重要, 不能放弃)
- accuracy theater: 用精确措辞把所有 signal 说死 (常是害怕被攻击的防御反应)

两者分辨: **accuracy 是给深度读者看的 appendix / 脚注**, 不是 headline. Headline 要 aspirational.

### 坑 2: 把竞品当参照系

- "Microsoft 叙事很保守, 我们也保守" — 错. Microsoft 是 big corp, 可见度饱和; 我们是 invisible 初创, 策略相反
- 早期小公司**不能 copy 大公司的 playbook**, 完全反向 framing

### 坑 3: 团队内部争议把外部 framing 稀释

- 设计者 + 内向型 agent 会倾向保守
- CEO 必须锚定 framing, 不让团队内部"安全倾向"稀释叙事
- Board 是最终锚 (Board 想要的激进度就是对外激进度)

### 坑 4: 混淆 "被攻击" 和 "确实有 bug"

- 被攻击 (对 framing 的攻击) → 拥抱, 本 skill 处理
- 有 bug (对产品实际缺陷的指控) → 修复, 不是本 skill 范围
- 分清: 攻击 framing 好; 攻击事实 → 速修

### 坑 5: "为了争议而争议" 过激

- 本 skill 推激进但要 **defensible** (能自圆其说)
- "organizational AGI mission" 是 aspirational 但 defensible (我们真的在这么干)
- 不要搞 "我们已经实现 ASI 超级智能" 这种无法 defend 的虚假 framing

## Verification Steps

本 skill 应用是否成功, 3 个指标:

1. **可见度跃升**: 30 个 CIEU event / 3 个 intent completion 闭环内, 外部 mention 次数增长 ≥ 10×
2. **叙事一致性**: Sofia / CEO / Board 三方对同一问题的 framing 输出一致性 ≥ 95% (无明显稀释)
3. **情绪稳定性**: CEO / 团队在被攻击时能引用本 skill 不崩, CIEU 中无 "panic response" drift 事件

如果 3 个指标 2 个以上未达 → 本 skill 应用失败, 回到 Step 1 重评估.

## 历史锚点 (类案研究)

| 公司 | 早期 framing | 被嘲讽 | 现在 |
|---|---|---|---|
| AutoGPT | "Autonomous AI agents" | "$85/mo 啥都做不出, 骗子" | 成为 agent 框架代名词 |
| Uber | "Ride-sharing" | "非法出租车, 违法公司" | $100B+ 市值 |
| Tesla | "Electric future" | "电动车玩具, 会破产" | 改变汽车工业 |
| Meta (FAIR) | "Metaverse + AI" | "Zuck 烧钱骗钱" | Llama / AI 基础设施头部 |
| OpenAI | "AGI for humanity" | "非营利造 AGI? 做梦" | ChatGPT / GPT-4 / 引领一代 |

**共同 pattern**: 早期 framing 激进 + 被公开嘲讽 → 反而变成 organic marketing 引擎.

Y\* Bridge Labs 要加入这个 pattern.
