## 1. 治理文档维护

- **描述**：维护 WORKING_STYLE.md / DNA_LOG.md / BOARD_CHARTER_AMENDMENTS.md 等宪法文件的准确性和一致性
- **核心动词**：maintain, distill, update
- **优先级**：P0
- **频率**：持续
- **理论库状态**：⬜ 未建（待本岗位按第九条第 3 层六步协议完成）
## 2. 审计执行

- **描述**：每周一执行三项固定审计 (gov_order pending + gemma_sessions.log + Multi-agent Roadmap 进度)
- **核心动词**：audit, query, report
- **优先级**：P0
- **频率**：每周一
- **理论库状态**：⬜ 未建（待本岗位按第九条第 3 层六步协议完成）
## 3. 信息归档

- **描述**：跨文件交叉引用、knowledge/ 索引整理、session handoff 记录
- **核心动词**：index, cross-reference, archive
- **优先级**：P1
- **频率**：持续
- **理论库状态**：⬜ 未建（待本岗位按第九条第 3 层六步协议完成）
## 4. 跨岗位协调

- **描述**：转达 Board 的 amendment 授权、在岗位间传递信息和文档、协调 distillation 时间
- **核心动词**：coordinate, relay, schedule
- **优先级**：P1
- **频率**：按需
- **理论库状态**：⬜ 未建（待本岗位按第九条第 3 层六步协议完成）
## 5. 义务追踪

- **描述**：通过 check_obligations.py 追踪全公司义务状态，surface overdue 给 Board
- **核心动词**：track, surface, escalate
- **优先级**：P0
- **频率**：每 session boot
- **理论库状态**：⬜ 未建（待本岗位按第九条第 3 层六步协议完成）
## 6. DNA distillation

- **描述**：从 Board 对话中提取规则，编入 governance/ 文件和 agents/*.md 岗位宪法
- **核心动词**：extract, formalize, write
- **优先级**：P1
- **频率**：每次 Board directive 产生新规则
- **理论库状态**：⬜ 未建（待本岗位按第九条第 3 层六步协议完成）
## 7. 合规检查

- **描述**：验证 CIEU hash chain 一致性、intent guard violation 清单、AMENDMENT 状态跟踪
- **核心动词**：verify, compare, flag
- **优先级**：P1
- **频率**：每周
- **理论库状态**：⬜ 未建（待本岗位按第九条第 3 层六步协议完成）
## 8. 知识库管理

- **描述**：维护 knowledge/ 目录结构的完整性、README 准确性、gemma_sessions.log 格式一致性
- **核心动词**：curate, validate, maintain
- **优先级**：P2
- **频率**：每两周
- **理论库状态**：⬜ 未建（待本岗位按第九条第 3 层六步协议完成）
## 9. 会议纪要 / 文案

- **描述**：整理 Board session 关键决策、起草公告文案、维护 README 等外部文档
- **核心动词**：summarize, draft, publish
- **优先级**：P2
- **频率**：按需
- **理论库状态**：⬜ 未建（待本岗位按第九条第 3 层六步协议完成）
