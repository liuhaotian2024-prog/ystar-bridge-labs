# Y*gov 系统收敛与反拉扯工作原则 v1

> 本文件是内部工作宪法。目标：防止Y*gov因局部优化而被反复拉扯，确保系统阶段性收敛。

## 全局不变式

1. **本体不变式**：Y*gov是治理内核，不是某个垂直场景产品
2. **架构不变式**：底座先于三条线，Foundation拥有基础主权
3. **主权不变式**：compiler/provider/scope是正式唯一入口
4. **路由不变式**：Path B→Bridge→GovernanceLoop是正式回流路径
5. **工程不变式**：少硬编码、IO无关、策略外提、参数化
6. **证据不变式**：所有关键治理动作进入证据链
7. **因果不变式**：Pearl L1/L2/L3属于Foundation，不归某条线私有

## 单阶段单主线

每阶段只有一个系统主线。所有任务服务该主线。不服务主线的任务延期。

## 任务三分类

- **收权型**：把分散主权收回底座（最高优先级）
- **强化型**：增强某模块能力（必须服从主线）
- **整洁型**：降低技术债（收权完成后优先级上升）

## 核销规则

讨论前必须先核销：已完成/部分完成/未完成。禁止模糊表述。

## 局部建议准入检查

1. 职责检查：是否让模块做不该做的事？
2. 边界检查：是否侵占别的层职责？
3. 复杂度检查：有没有更轻的方案？
4. 收敛检查：让系统更收敛还是更发散？

## 停止条件

没有停止条件的任务不开工。

## 工作口径

> Y*gov当前阶段的核心任务，不是无限增强局部能力，而是保证系统收敛。
> 任何局部优化都必须服从全局不变式、当前主线、核销纪律与停止条件。
