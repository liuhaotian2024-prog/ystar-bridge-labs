- Human CEO uses dashboards for instant progress judgment

**Goal**: CIEU-driven progress dashboard that automatically calculates % done for each target.

---

## Solution Architecture

### Core Components

1. **`scripts/goal_progress.py`** — Main dashboard generator
   - Reads `reports/priority_brief.md` YAML frontmatter (today/week/month targets)
   - For each target's `verify` field, extracts keyword markers
   - Queries `.ystar_cieu.db` for matching CIEU events (lookback: 48h/7d/30d)
   - Calculates completion % = (markers_hit / total_markers) × 100
   - Generates markdown dashboard with ASCII progress bars

2. **`reports/goal_progress.md`** — Live dashboard output
   - Auto-generated, DO NOT manually edit
   - Shows today/week/month targets with progress bars
   - Updates every 15 minutes via cron

3. **`scripts/goal_progress_statusline.sh`** — Statusline integration
   - Extracts today's first target % for statusline display
   - Color-coded: red (<50%), yellow (50-79%), green (≥80%)
   - Output format: `[Goal:37%]`

4. **`scripts/goal_progress_cron.sh`** — Auto-refresh daemon
   - Cron job: `*/15 * * * *`
   - Regenerates dashboard every 15 minutes

5. **`tests/test_goal_progress.py`** — 9 comprehensive tests
   - YAML parsing, marker extraction, progress bar rendering
   - CIEU query integration, lookback window filtering
   - Full E2E dashboard generation

## CIEU Event Matching Strategy

### Marker Extraction

From `verify` string, extract alphanumeric + underscore tokens:
- "Maya commit + RLE on_cieu_event" → ["Maya", "commit", "RLE", "on_cieu_event"]
- "17 base + 10 tests 全绿" → ["17", "base", "10", "tests"]
- "pytest pass" → ["pytest", "pass"]

### CIEU Query

For each marker, search in `.ystar_cieu.db` table `cieu_events`:
- Columns: `decision`, `event_type`, `file_path`, `command`, `task_description`, `agent_id`
- Lookback: 48h (today), 7d (week), 30d (month)
- Match: `LIKE '%marker%'` (case-insensitive via SQLite collation)

### Completion Calculation

```
completion_% = (markers_with_hits / total_markers) × 100
```

Example:
- Verify: "pytest pass + commit + Maya" (4 markers: pytest, pass, commit, Maya)
- CIEU hits: pytest=5, pass=5, commit=0, Maya=3
- Completion: 3/4 = 75%

## Known Limitations

1. **Keyword matching is fuzzy** — "commit" matches both "git commit" and "commitment"
   - Mitigation: Use specific markers like "commit_hash" or "git_commit_msg"

2. **No semantic understanding** — Can't detect "test failed" vs "test passed"
   - Mitigation: Use boolean CIEU event `passed=1` field (TODO: future enhancement)

3. **No target dependency graph** — Can't detect if target A blocks target B
   - Mitigation: CEO manually sequences targets in priority_brief.md

4. **15-min refresh latency** — Not real-time
   - Mitigation: Run `python3 scripts/goal_progress.py` manually for instant update
