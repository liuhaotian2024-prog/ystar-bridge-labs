# Root Cause Analysis: PreToolUse Hook Silent Failure in Interactive Sessions

**Date**: 2026-04-02
**Investigator**: CEO Agent (Board-directed investigation)
**Severity**: P1 — Governance blind spot, 10+ hours of unaudited Board activity
**Status**: ROOT CAUSE IDENTIFIED + FIX VERIFIED

---

## Symptom

| Mode | Hook fires? | CIEU records? |
|------|-------------|---------------|
| `claude -p` (non-interactive) | Yes | Yes — session `ystar-company_0a84f3c6` |
| `claude` (interactive Board) | **No** | **Zero for 10+ hours** |

## Root Cause

**Settings merge shadowing**: Project `.claude/settings.json` (created 2026-03-26) shadows global `~/.claude/settings.json` hooks in interactive mode.

### The Configuration Stack

| Priority | File | Has `hooks`? | Has `permissions`? |
|----------|------|-------------|-------------------|
| 1 (lowest) | `~/.claude/settings.json` | **Yes** — `from_agents_md(confirm=False)` | No |
| 2 | `.claude/settings.json` | **No** | Yes — `Bash(*), Read(*), Write(*), Edit(*)` |
| 3 (highest) | `.claude/settings.local.json` | **Yes** — `from_agents_md_multi` | Yes |

### What Happened

1. `.claude/settings.json` was created on 2026-03-26 with only `permissions`, no `hooks`
2. Claude Code's settings merge behavior: when a project-level settings file exists, it takes precedence over global settings at the **key level**
3. **Critical bug/behavior**: In interactive mode, the presence of project `.claude/settings.json` prevented global `~/.claude/settings.json`'s `hooks` key from being inherited — even though the project file didn't define `hooks` at all
4. In `-p` mode, the settings resolution followed a different code path that DID inherit global hooks (confirmed by CIEU records from daemon sessions)

This matches the known Claude Code issue [anthropics/claude-code#19487](https://github.com/anthropics/claude-code/issues/19487) regarding settings merge behavior.

### Why `-p` Mode Worked

The agent daemon (`scripts/agent_daemon.log`) launches `claude -p` sessions. These sessions:
- Successfully inherited global hooks from `~/.claude/settings.json`
- Created CIEU records with session_id `ystar-company_0a84f3c6`
- Used `from_agents_md(confirm=False)` (global hook variant)

### Why Interactive Mode Failed

Board sessions (`claude` direct launch) in this project directory:
- Loaded project `.claude/settings.json` (priority 2)
- This shadowed the global settings, effectively dropping the `hooks` configuration
- Result: PreToolUse hook never fired, zero CIEU records for all interactive work

## Fix Applied

`.claude/settings.local.json` was created (2026-04-01 22:19:25 ET) with hooks at **highest priority** (level 3), bypassing the merge issue entirely.

## Additional Findings

### 1. Hook Output Format Non-Standard (Low Risk)
The hook returns `{}` on allow. Claude Code expects either:
- Exit code 0 alone (allow)
- JSON with `hookSpecificOutput.permissionDecision`

The `{}` format works because exit code 0 = allow, but it's not the documented format. Recommend aligning with spec.

### 2. Matcher Field Correct
`"matcher": ""` matches all tools — this is equivalent to `"*"`. No issue here.

### 3. Session ID Not Unique Per Session
Both daemon and interactive sessions produce `ystar-company_0a84f3c6` as session_id (derived from project path hash). This makes it impossible to distinguish interactive vs non-interactive CIEU records. Recommend passing a unique session identifier.
