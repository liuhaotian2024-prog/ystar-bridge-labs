## Hacker News

### Limits
- Title: 80 characters max
- No body text (link posts only, or text posts for Ask HN)
- Comments: plain text only, no markdown, no HTML

### What Works
1. "I built X and it did Y" (results-first)
2. "Why X failed" (honest postmortems)
3. Technical deep-dives with real benchmarks
4. Counterintuitive insights backed by data
5. Show your work, share the code

### What Gets Killed
- Marketing hype, "revolutionary", "game-changing"
- Vague claims without data
- Self-promotion without disclosure

### Rules
- One post per account per day
- Ask HN > Show HN for non-project updates
- Comments must add value

## LinkedIn

### Limits
- Post: 3000 characters max
- Optimal: 800-1000 words (NOT the 2000+ we wrote today — CASE-006)
- Supports line breaks and emojis

### What Works
1. "I made a mistake and what I learned"
2. Real metrics + charts + specific numbers
3. Founder journey (authentic, not polished)
4. Contrarian take with reasoning
5. Thread-style numbered lists

### Rules
- 3-5 posts per week optimal
- Algorithm favors consistent posting

## Telegram Channels

### Limits
- Optimal: 300-800 characters per message
- Supports rich text, links, media

### What Works
- Direct, bullet-pointed updates
- Real-time events (governance denials, commits)
- Short, shareable standalone messages

## Twitter/X

### Limits
- 280 chars (free), longer with premium
- Optimal: 100-150 chars for max engagement

### What Works
- Bold hot takes
- Threads for longer content
- Quote-tweet pattern for engagement
