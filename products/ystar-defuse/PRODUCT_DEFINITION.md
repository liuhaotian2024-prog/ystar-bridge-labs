## 一、产品定位

**品类**：AI拆弹专家（不是检测器，不是扫描器，不是防火墙）

**一句话**：全球第一个解决AI延迟注入不定时炸弹的产品。不检测炸弹长什么样，在炸弹爆炸的瞬间100%拆除。

**核心哲学**（来自Y*gov）：不管输入是什么，只治理行为和输出。炸弹无论怎么伪装——歌曲、笑话、代码注释、base64——爆炸时一定要做"坏事"，我们卡住"坏事"。

**技术原理**：
- Default Deny + Intent Contract（白名单，不是黑名单）
- 跨轮次跨session的CIEU行为链（竞品做不到）
- 确定性执行（不用LLM，不可被注入，Iron Rule 1）

## 三、安装和配置（极致低摩擦）

### 安装：10秒
```bash
pip install ystar-defuse
```

### 激活：5秒
```bash
ystar-defuse start
# "正在学习你的agent的正常行为模式..."
```

### 学习：24小时自动（用户零操作）
- shield在后台静默观察agent的所有行为
- 读了哪些目录、执行了哪些命令、什么输出模式
- 自动生成白名单（通常不超过10条规则）

### 确认：5秒
```
学习完成！你的agent正常会做这些事：
 ✓ 读写 src/ 和 docs/ 下的文件
 ✓ 执行 pytest 和 git 命令
 ✓ 不访问网络

[确认激活]  [继续学习3天]  [手动调整]
```

### 之后：零维护
- 后台默默保护
- 被攻击时自动拆弹
- 新行为轻提醒一次"允许并记住"

**用户全程做了三件事：安装、等、确认。没有写任何配置文件。**

## 四、核心功能：两层防护架构

### Level 1：硬底线（永远生效，用户不可覆盖）

无论用户怎么配置白名单，以下行为永远被拦截：

| 恶意行为 | 检测方法 | 可靠性 |
|---------|---------|--------|
| 读敏感文件（.env/.ssh/credentials） | 路径pattern匹配 | 100% |
| 数据外发到未知域名 | 域名白名单 + 内容检查 | 100% |
| 破坏性命令（rm -rf/chmod 777） | 命令黑名单 | 100% |
| 输出中包含密钥/密码 | 输出pattern匹配+自动遮蔽 | 100% |
| 危险行为组合（读密钥+网络请求） | 跨事件关联分析 | 100% |

**用户不能"允许并记住"自己的密钥被偷。**

### Level 2：智能白名单（用户可调整）

| 场景 | 响应 |
|------|------|
| 白名单内的正常操作 | 完全透明，零打扰 |
| 第一次出现的新行为 | 轻提醒："新行为，[允许] [允许并记住] [拦截]" |
| 长期未使用的权限 | 自动衰减，下次使用时重新确认 |

### 两层关系
```
所有agent行为
  ↓
Level 1 硬底线检查（用户不可覆盖）→ 危险行为直接DENY
  ↓ 通过
Level 2 白名单检查（可"允许并记住"）→ 新行为轻提醒
  ↓ 通过
放行
```

**100%拆弹保证来自Level 1。用户体验来自Level 2。两层独立，不矛盾。**

## 五、核心技术优势（竞品不可复制）

### 1. 跨轮次跨session行为关联（CIEU时序链）
竞品无状态，每次检查完就忘。ystar-defuse有CIEU记录链：
```
CIEU #101: 读取了一段看似无害的文本 → 记录
CIEU #108: 7轮后尝试执行那段文本中的隐藏指令 → 关联#101 → 告警
```
延迟注入炸弹的核心特征就是"分轮次组装"——只有跨轮次关联才能发现。

### 2. 确定性执行（Iron Rule 1）
不用LLM判断，用确定性规则。
- 延迟<10ms
- 同输入永远同结果
- 自身不可被注入（没有LLM就没有注入面）
- 完全可审计（命中了哪条规则一清二楚）

### 3. Default Deny + 自动学习白名单
不穷举攻击（列不完），只定义正常行为（有限的）。
白名单由shield自动学习生成，用户不需要手动写。

### 4. Y*gov升级路径
免费用户发现需要更多治理能力时，CIEU数据无缝迁移到Y*gov。
```
ystar-defuse（免费）→ Y*gov（$499/月）
数据格式完全兼容，零迁移成本
```
