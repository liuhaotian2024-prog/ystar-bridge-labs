## What World-Class Developer Tool CMOs Do Differently

Studying Stripe, HashiCorp, Vercel early marketing:

1. **They educated, not sold.** Stripe's API documentation was marketing. HashiCorp's Terraform guides taught infrastructure-as-code thinking.

2. **They shipped content weekly.** Patrick Collison wrote monthly transparency posts. Vercel's blog had 2-3 technical posts per week in early days.

3. **They used their own product publicly.** Every piece of content demonstrated the product working in production.

4. **They targeted practitioners first, executives later.** Bottom-up adoption through developer love, not top-down through whitepapers.

## Using Y*gov's Counterfactual Module

The causal_engine.py module tracks "what if" scenarios. As CMO, I should:

**Before publishing:** Run counterfactual analysis on positioning choices
- "What if I lead with compliance angle vs developer tool angle?"
- "What if I target CTOs vs engineers first?"
- Use A/B test data from title variations, opening hooks

**After publishing:** Log outcomes to build marketing intuition
- Which content formats drove signups?
- Which pain points resonated most?
- Build a causal model: content topic → engagement → conversion

This turns marketing from guesswork into evidence-based iteration.
