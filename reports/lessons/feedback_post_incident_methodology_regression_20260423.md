# Lesson — Post-incident methodology regression (firefighter mode)

## 规则

任何 post-incident autonomous work mode 启动 (hook deadlock 解锁后 / daemon crash 修复后 / canonical rule repair 后 / 其他事故闭环后) **必先过 methodology stack 全量 checklist**, 不待 Board 提醒才补.

## 必过 checklist (dispatch 首 subagent 前)

- [ ] **M-tag** 对齐 (M-1 Survivability / M-2a 不乱做事 / M-2b 不能不做事 / M-3 Value Production)
- [ ] **14 原则 Top-5 相关条** (P-3 反事实 / P-4 empirical / P-7 传导链 / P-12 先查后造 / P-14 诚实)
- [ ] **8-module 认知** per-task (M1 反射 / M2 记忆 / M3 资料 / M4 OODA / M5 方法论 / M6 错误监控 / M7 主动 / M8 反思)
- [ ] **4-factor engineer culture** (model × role × culture × prompt high-stakes framing)
- [ ] **6-问 reply-time** (L-tag / 5-tuple / 定量诚实 / plan vs done / present tense / 无选择题)
- [ ] **8-cascade** (新 entity charter/registry/boot/dispatch/hook/CIEU/FG-scope/pre-auth)
- [ ] **Baseline empirical snapshot** (全系统扫描器 + CIEU SQL + traditional Channel 2) 在 Wave 前跑
