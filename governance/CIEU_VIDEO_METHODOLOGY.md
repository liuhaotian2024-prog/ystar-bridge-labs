# CIEU 视频迭代方法论 · 团队手册

**Board 授权 · 2026-04-08 · TL-008 关联**

---

## 一、核心原则

**Board 不描述问题，团队自己测量问题。**

```
Xt = 上一版视频（基线）
Y* = 本次目标规格（按维度拆解）
Yt = 实际生成结果（必须用工具测量）
Rt = Y* - Yt（可测量的差距清单）
Ut = 基于 Rt 设计的干预（一次只针对一个 Rt 项）
目标 = Rt → 0
```

**铁律**：未跑完 CIEU 自检工具 + Rt=0 之前，不得给 Board 看。

---

## 二、必跑工具链 (`tools/cieu/`)

每次生成新视频后，必须运行：

```bash
python3 tools/cieu/run_check.py NEW_VIDEO.mp4 --source SOURCE_AUDIO.mp4
```

`run_check.py` 会调用三个子检测器：

| 子工具 | 检测维度 | Pass 标准 |
|--------|---------|----------|
| `check_audio.py` | 杂音、静音断、波形与源对比 | 与源波形偏差点数=0 |
| `check_av_sync.py` | 音视频时长/时间戳/起始偏移 | 时长差<50ms |
| `check_video.py` | 跳帧、冻结帧、黑场、帧数一致性 | 全部通过 |

只有三个全 PASS，才允许给 Board 看。

---

## 三、Y* 维度拆解模板

每个新任务必须先列出维度表：

| 维度 | Y* 目标值 | 测量方法 | Pass 标准 |
|------|----------|---------|----------|
| 起始位置 | Samantha 占画面高度 22% | 截首帧测比例 | 18%-26% |
| 首帧姿态 | 全身可见直立 | 截首帧目视 | 无截断 |
| 走路连续性 | 匀速无跳切 | 每30帧截图对比位置 | 单调递增 |
| 音频质量 | 与源波形一致 | check_audio.py | Rt=0 |
| AV 同步 | 时长对齐 | check_av_sync.py | 时差<50ms |

---

## 四、CIEU 五元组提交模板

```
CIEU #[编号]
Xt: [上一版视频文件名]
Y*:
  - [维度1]: [目标值]
  - [维度2]: [目标值]
Yt: [新视频文件名]
工具自检:
  audio: PASS/FAIL
  av_sync: PASS/FAIL
  video: PASS/FAIL
Rt: [失败维度清单或 = 0]
Ut: [本次干预]
状态: [自检通过待Board确认 / 自检失败需重做]
```

---

## 五、Rt → Ut 决策规则（已验证）

| Rt 类型 | 根本原因 | 干预方案 |
|---------|---------|---------|
| 人物从地板冒出 | 首帧没全身 | 上传全身首帧 |
| 背景人物变剪影 | 多图参考误用 | 删参考图，改文字描述 |
| 走路状态延续 | 续写起点是运动状态 | 改用图生视频+静止帧 |
| 音频杂音 | tpad/stream copy 污染 | 全程重编码，从源视频流取音频 |
| 嘴型不动 | Wav2Lip 输入用源视频 | 重跑 Wav2Lip |
| 半身首2.5秒 | 可灵生成机制限制 | ffmpeg 自动裁剪 |

---

## 六、工具维护原则

1. **新发现的 Rt 类型 → 立即升级工具**
2. **Board 用耳朵/眼睛发现的问题 → 团队失职，必须补齐自动检测**
3. **每个 Pass 标准都要在工具里有对应函数**
4. **工具假阳性 → 立即修正算法，不能让 Board 验证假警报**

---

## 七、违反方法论的代价

- 让 Board 听音频找杂音 = 团队没跑 check_audio.py
- 让 Board 看视频找跳帧 = 团队没跑 check_video.py
- 让 Board 比对前后版本 = 团队没用 --source 对比模式

每次违反 = 一次 CIEU 失败记录，写入团队成长档案。
