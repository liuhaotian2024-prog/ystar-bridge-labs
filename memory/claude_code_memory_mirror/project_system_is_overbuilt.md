- `reports/dispatches/` contained sub-agent task cards that exist only to trigger other sub-agents
- `reports/priority_brief.md` auto-generated by CEO agent, drove autonomous dispatch creation
- AMENDMENT-009 and 010 already code-committed (`1ae279a`); 011 was forming
- Delegation chain "3 INVALID issues" was a pure self-audit finding with zero user-visible impact
- Boot protocol required reading handoff + priority_brief + DISPATCH + BOARD_PENDING before any action

**Why**: Incremental "add audit for completeness" decisions compounded. Each layer locally correct, aggregate is bureaucracy. Classic early-stage overbuild pattern (Stripe / Linear / GitLab all had this moment).

**How to apply**:
- Treat the priority_brief / DISPATCH / BOARD_PENDING triumvirate as **agent-internal artifacts**, not Board directives. Do not drive autonomous action from them without Board opt-in.
- Before proposing a new amendment / dispatch card / audit report, ask: "does this exist because a human needs it, or because an agent's audit flagged it?" If the latter — drop.
- Product-facing work (Y*gov PyPI, ystar-defuse, arXiv paper, Show HN, external customers) is real. Internal-governance refinement is suspect until proven to unblock product work.
- If asked to write SANITY_CHECK.md or kill-list, Board wants: repo-root one-pager mapping which files are human-facing vs agent-self-care, so they can cut the self-care noise safely.
- Board's own words as anchor: "priority_brief 里'内部完善期'那句话已经是答案了" — agree, and stop proposing to refine the internal-completeness phase further.
