## 行为规则
sub-agent 后台并行跑时，CEO 本线必须立刻推进下一项工作。**禁止**说"等 X 回来"、"等通知"、"仍在跑"这类被动语态。

**Why:** 被动语态暴露 CEO 线空转——浪费 CEO 本线可并行做的工作（read、决策、其他派活、知识回写）。Board 要的是组织级并行，不是一条线串行等待。

**How to apply:**
- sub-agent 派出去后，CEO 本线立刻推进下一项（不等 notification）
- notification 到达时做 triage + 下一轮派工，不是终点
- 任何"等"字出现前先问：本线能同时做什么？

## ⚠️ 第十一条原文澄清（防止再次误用）

**第十一条 ≠ "后台 N 路并列"**。这是 framing 口语，不是条文。

**第十一条原文**（governance/WORKING_STYLE.md:783-880）= **Autonomous Mission 方法论**：
- 阶段一 认知建构（执行前必须完成 + Board 批准）：层 0 意义 / 层 1 目标 / 层 2 假设 / 层 3 理论 / 层 4 Benchmark / 层 5 相关性 / 层 6 案例提炼 / 层 7 能力边界
- 阶段二 执行：层 8 方案设计 + 层 9 执行（CIEU 记录）+ 伦理强制检查
- 阶段三 观察：48h 内定量+定性+假设验证
- 阶段四 迭代：4 触发条件 + 螺旋上升

Board 审阅的是**认知深度**不是计划。

**用法**：
- ✅ "我对 X Mission 走第十一条 7 层认知建构" = 走方法论
- ❌ "启动第十一条，后台 N 路并列" = 把口语框架当条文（2026-04-13 Board 当面纠正）
- 描述并行派活就直接说"派 A/B 后台并行，本线同时推 C"
